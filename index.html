Create a New Project

Navigate to GitHub.com and click Start a project

Create a New Repository

In the form that appears, enter the Repositpry name and a Description.

For the repository name, make sure you enter exactly your username followed by .github.io

In the description field, enter any description you see fit. I entered "My Portfolio".

There's no need to modify any of the other options, so click Create repository

Open in GitHub Desktop

After GitHub has created your repository, you now need to get that repository onto your local machine. The easiest way to do so is using GitHub Desktop, GitHubs GIT GUI. If you don't have it, you can download it here.

On the page that appears, under the Quick Setup area, click Set up in Desktop. A popup will appear, click Open GitHubDesktop.exe when prompted.

Clone Repository

After GitHub Desktop opens, a popup will appear with the URL to your repository as well as a local path

The local path is usually not exactly where you want it to be, so click Choose... and choose a different folder. This is where all of your code will be saved, so choose wisely.

Double-check Repository Folder

Before we create the website, it's best to double-check the clone was successful. Navigate to the local path you cloned the repository to, and check for a .git folder.

If you do not see the folder, ensure you're able to see dot folders. In the File Explorer, click View then check the Hidden items checkbox.

If you see the .git folder, move onto the next step. If you do not, redo step 4.

[Optional] Download Template

Note: The following 3 steps are optional. If you already have a website, move to Step 9

Now, you'll need a website. An easy way to get started is to use a template. I'm using a template from startbootstrap.com, specifically the Stylish Portfolio template.

Once you've found a template you fancy, click the Download button.

[Optional] Extract Template

After the template has downloaded, extract it. Make sure to extract it to the root folder of your repository folder (the same folder with the .git folder.

[Optional] Personalize Template

Next, open the index.html file in the editor of your choice and make any personalization change to the file. I chose to edit the <title> element.

Commit

Now it's time to put your website on GitHub. Open GitHub Desktop again.

You will see a long list of added files to the left, if you don't see any files, either GitHub is still gathering them or you did not add your files to the correct directory.

In the bottom left, you'll see a small form with input for a Commit title and Description. Enter a title, I wrote "Initial Commit of Portfolio". There's no need to enter a description for this commit, but if you want to, you can.

Publish Commit

You've committed your changes, but they're still not on GitHub, they remain on your local system. In order to push the to GitHub, click the Publish branch button.

Confirm Publish

After GitHub Desktop has completed publishing, confirm that everything went as expected by going back to github.com and refreshing the page.

You will now see all of your website files.

GitHub Pages

Click the Settings tab on GitHub. Scroll down to the GitHub Pages area. Here, you should see a confirmation message that your site has been published.

If you do not see this message, scroll back to the top of the settings page and confirm that the repository name is exactly the same as your username.

Click the link to go to your new portfolio website.

Marvel at your Portfolio

If all went well, you will now be looking at your flashy new portfolio website!
